## ✨ Функционалност

| Анализ | Описание |
|--------|----------|
| **Месечно сравнение** | Разход, обем и WAP за всеки месец — YoY делта |
| **ABC анализ** | Класиране на SKU по приход (A = топ 80%, B = 80–95%, C = остатък) |
| **WAP по артикул** | Претеглена средна цена EUR/кг по артикул × месец |
| **Доставчици** | Пазарен дял, месечна разбивка, YoY сравнение |
| **Ценова волатилност** | CV% по артикул с рейтинг (висока / умерена / стабилна) |
| **Авто-верификация** | Сумите от хедърите се проверяват срещу артикулните редове |

**Технически особености:**
- 🔍 Автоматично открива годините от данните (time-agnostic)
- 🏗️ Автоматично разпознава layout-а на файла (2025 vs 2026 формат)
- ⚠️ Обработва `ТРАНСФЕРЕН ПРОТОКОЛ` и `ФАКТУРА` под-документи
- 🟡 Маркира артикули с EUR = 0 (реална доставка без цена)
- 📊 15 000+ Excel формули в изходния файл — всичко е проследимо

## 🚀 Инсталация

### 1. Клонирай репозитория

```bash
git clone https://github.com/<username>/fruit-veg-analyzer.git
cd fruit-veg-analyzer
```

### 2. (По избор) Създай виртуална среда

```bash
python -m venv venv

# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
```

### 3. Инсталирай зависимостите

```bash
python -m pip install -r requirements.txt
```

> **Windows:** Ако `pip` не се разпознава, използвай `python -m pip` вместо `pip`.

## 💻 Употреба

### Основна команда

```bash
python analyzer.py --baseline <файл_2025.xlsx> --ytd <файл_2026.xlsx>
```

### Всички аргументи

```
python analyzer.py [-h] --baseline FILE --ytd FILE [--output NAME]

Задължителни:
  --baseline, -b    Excel файл с историческата (базисна) година
  --ytd,      -y    Excel файл с текущата YTD година

Незадължителни:
  --output,   -o    Базово име на изходния файл (без .xlsx)
  --help,     -h    Показва помощ
```

### Примери

```bash
# Минимален пример
python analyzer.py -b "Плодове 2025.xlsx" -y "Плодове 2026.xlsx"

# С персонализирано изходно ime
python analyzer.py --baseline "2025.xlsx" --ytd "2026.xlsx" --output "Отчет_Q2_2026"

# С пълни пътища (Windows)
python analyzer.py \
  --baseline "D:\Данни\Плодове_2025.xlsx" \
  --ytd      "D:\Данни\Плодове_2026.xlsx" \
  --output   "D:\Отчети\Анализ_2025_2026"
```

### Специални редове (автоматично прескачани)

| Тип | Причина |
|-----|---------|
| `ТРАНСФЕРЕН ПРОТОКОЛ` | Вътрешен трансфер между складове |
| `ФАКТУРА` | Дублиращ счетоводен документ |
| Мерна единица ≠ `кг` | Напр. `бр` (брой) |

> ⚠️ **Артикули с EUR = 0** се включват в обема (кг), но се изключват от WAP.
> В листа `ITEMS_2026` те са маркирани с жълт фон.

## 📊 Структура на изходния отчет

| # | Лист | Описание |
|---|------|----------|
| 1 | `RAW_{год}` | Фактурни хедъри — единственият верен source на EUR суми |
| 2 | `RAW_{год}` | Фактурни хедъри за YTD годината |
| 3 | `ITEMS_{год}` | Артикулни редове — baseline (кг и WAP) |
| 4 | `ITEMS_{год}` | Артикулни редове — YTD (жълто = EUR=0) |
| 5 | `⚙️ Assumptions` | Параметри: BGN/EUR курс, ABC прагове, CV прагове + авто-верификация |
| 6 | `📊 Monthly Summary` | Месечно сравнение: разход, обем, WAP, YoY делта |
| 7 | `🏢 Доставчици` | Пазарен дял, месечна разбивка, YoY по доставчик |
| 8 | `🔤 ABC Analysis` | Класиране на артикули по приход |
| 9 | `📈 WAP по SKU` | Претеглена средна цена по артикул и месец |
| 10 | `⚡ Волатилност` | Ценова волатилност CV% по артикул |

## 🎨 Цветово кодиране

### Формули в Excel

| Цвят | Значение |
|------|----------|
| 🔵 Синьо | Входна стойност от изходния файл (не се променя) |
| ⚫ Черно | Формула — изчислява се автоматично |
| 🟢 Зелено | Препратка към друг лист (cross-sheet SUMIF / SUMPRODUCT) |
| 🟡 Жълт фон | Артикул с EUR = 0 в `ITEMS_2026` |

### ABC класове

| Клас | Цвят | Критерий |
|------|------|----------|
| A | 🟢 Зелен | Топ 80% от разхода |
| B | 🟡 Жълт | 80–95% от разхода |
| C | 🔴 Червен | Остатъчни 5% |

### Ценова волатилност

| Рейтинг | Критерий |
|---------|----------|
| 🔴 Висока | CV > 30% |
| 🟡 Умерена | CV 15–30% |
| 🟢 Стабилна | CV ≤ 15% |

> Праговете се управляват в `⚙️ Assumptions` — промяна там се отразява
> автоматично в целия отчет.

## 🏢 Добавяне на нов доставчик

Скриптът открива новите доставчици **автоматично** — не е нужна промяна на кода.

По избор можеш да зададеш цвят за реда в отчета.
В `analyzer.py` намери (~line 95):

```python
VENDOR_COLORS = {
    "РЕГОЛИТ ЕООД":       "D4EDDA",
    "АКВАТЕК ООД":         "D1ECF1",
    "ПИКАНТ ТРЕЙД ООД":    "FFF3CD",
    "ГОЛДЪН ПЛОД-ЕООД":    "FDE8E8",
    "ВЕЖЕН 2010  ЕООД":    "F3E5F5",
}
```

Добави нов ред:

```python
    "НОВ ДОСТАВЧИК ЕООД":  "E3F2FD",  # светло синьо
```

Ако не добавиш — доставчикът се появява с **бял фон**, всичко друго работи нормално.

## 🐛 Известни проблеми и решения

### `'pip' is not recognized`

```bash
python -m pip install -r requirements.txt
```

### `No such file or directory: 'requirements.txt'`

CMD не е в правилната папка. При D: диск:

```
D:
cd "D:\пътят\към\папката"
```

```bash
dir *.xlsx
```

Провери точните имена и ги постави в кавички.
